# Audit Trail — Automated Work Tracking

## Purpose

Every session must produce a verifiable audit trail: what changed, why, when, and at what cost. This enables stakeholders to understand project progress without reading code or Engram.

## Rule (Always Active)

### Mandatory Session Outputs

Every session that produces code changes MUST generate:

| Artifact | Generated By | Location |
|---|---|---|
| Git context | `hooks/git-context-capture.sh` (Stop) | `.cognitive-os/sessions/{id}/git-context.json` |
| Session changelog | `hooks/session-changelog.sh` (Stop) | `.cognitive-os/changelogs/{session_id}.md` |
| Audit-enriched metrics | `hooks/audit-id-enricher.sh` (PostToolUse) | `session_id` field in cost-events.jsonl |

### Cross-Cutting Audit ID

All JSONL metric entries SHOULD carry these fields when available:

| Field | Source | Required |
|---|---|---|
| `session_id` | Env: `COGNITIVE_OS_SESSION_ID` | Yes |
| `sprint_id` | `sprint-status.yaml` | If sprint active |
| `change_id` | SDD pipeline state | If SDD active |
| `branch` | `git rev-parse --abbrev-ref HEAD` | Yes |

### Traceability Gate

Before running `/sdd-archive`, the orchestrator SHOULD run `/traceability-check` to verify:
- Every requirement has a spec reference
- Every spec has code implementing it
- Every code change has test coverage

Gaps are reported, not blocked (advisory).

### Sprint Changelog

When a sprint is active (`sprint-status.yaml` exists), session changelogs are automatically aggregated into `changelogs/sprint-{sprint_id}.md`.

## Enforcement

| Phase | Missing Audit Trail | Action |
|---|---|---|
| reconstruction | WARNING | Session continues, audit gap logged |
| stabilization | WARNING | Session continues, audit gap logged |
| production | WARNING | Session continues, reported in session summary |
| maintenance | WARNING | Session continues, reported in session summary |

Audit trail is never blocking — it captures after the fact, not gates before.

## Metrics

- `session-audit.jsonl` — one entry per session with git context summary
- `changelogs/{session_id}.md` — human-readable per-session
- `changelogs/sprint-{sprint_id}.md` — aggregated per-sprint

## Integration

| Component | Relationship |
|---|---|
| Session lifecycle | Git context captured at Stop, enricher runs on PostToolUse |
| Sprint system | Sprint ID threaded into all metrics when active |
| SDD pipeline | Change ID threaded when SDD is active |
| Cost tracking | cost-events.jsonl enriched with audit context |
| Engram | Session summaries complement changelogs (memory vs files) |

## Contextual Trigger

This rule is always active when the project-audit package is installed.
