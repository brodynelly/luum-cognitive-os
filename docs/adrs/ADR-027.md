# ADR-027: SO Slimming — Test Strategy, Context Overhead, Resource Consumption

## Status
ACCEPTED (2026-04-21) — WS1-WS3 shipped, included in v0.12.0 release. Implementation commits: 8dc4a6e, 9bd895b, 15d67eb. Resolved by ADR-027a. Originally proposed 2026-04-17.

## Context

luum-cognitive-os (the SO) has accumulated three classes of debt during
the reconstruction phase that now visibly harm orchestration throughput:

1. **Test verification strategy is unbounded.** Agents launched in parallel
   each inherit an acceptance criterion like `python3 -m pytest tests/unit/`.
   With 3 parallel agents per sprint, the full suite runs 3× redundantly.
   The Lote 4 run (2026-04-16, ~1 h wall-clock) spent the majority of its
   time re-running the same suite across agents instead of doing work.
2. **Context overhead per session and per agent-launch is opaque and large.**
   The global `~/.claude/CLAUDE.md` alone is ~24 KB (~6 K tokens) and loads
   on every session start; four `PreToolUse Agent` hooks fire on every
   agent launch with partially overlapping output (`blast-radius`,
   `scope-proportionality`, `project-gotchas`, `completeness-check`), and
   `self-install.sh` echoes a multi-line "installed/verified" banner on
   every `SessionStart` even when nothing changes.
3. **Resource consumption keeps growing silently.** JSONL metric files are
   append-only with no rotation; `.claude/settings.json` registers **27
   hook entries** total (with overlaps); every `PreToolUse Agent` invocation
   forks multiple shell + python sub-processes. The 49-minute symlinks
   debugging session on 2026-04-17 (commits `a59bf2e`, `57ed5cf`) was
   extended substantially by hook noise and tool-output filtering that
   masked where time was being spent.

This ADR is **planning + measurement only**. It produces: (a) a measured
baseline, (b) three decisions with feature-flagged rollbacks, (c) a carving
of implementation surface across three phases so the work can be parallelised
safely.

By inspecting the first ~50 lines of each hook's source:

| Pair | Overlap evidence |
|---|---|
| `blast-radius-detector.sh` ↔ `scope-proportionality.sh` | Both count changed files and warn on "too many files". `scope-proportionality` adds budget-vs-scope advice; `blast-radius` adds cross-service advice. ~60 % of heuristics overlap. |

**Key finding:** the literal `pytest tests/unit` is not defined in one

parallel agents all inherit the full suite.

## Decisions

### D1 — Test verification strategy

**Rule:** agents verify TARGETED (only the files they touched);
orchestrator verifies GLOBAL (full suite) ONCE after the last commit of
a sprint.

- Acceptance-criteria templates in skills MUST NOT hard-code
  `pytest tests/unit`. They use `{targeted_test_command}`, which the
  orchestrator (or the skill's resolver) maps to only the test files that
  cover the changed files.
- New section in `rules/agent-quality.md`: *"Agents verify targeted;
  orchestrator verifies global, once."*
- New `Stop` hook `hooks/global-verify.sh`: if any commits were made
  during the session, runs the full suite once.
- Gated by `cognitive-os.yaml: testing.global_verify_on_stop: true` and
  `testing.targeted_verify: true`.

### D2 — Context overhead

- `self-install.sh` already has SHA-based caching but still emits a
  banner on cache-hit. **Make banner silent on cache-hit**, print only on
  cache-miss or failure. Flag: `session.silent_self_install`.
- **Migrate `~/.claude/CLAUDE.md` to compact ref-key index** (~1.5 KB,
  ~30 lines), following the scheme already prototyped in
  `rules/RULES-COMPACT.md`. Full rule text loads on-demand via a
  `ref_key_loader`. Flag: `context.claude_md_compact`.
- **Preamble slimming.** The per-Agent-launch hook quartet
  (`blast-radius`, `scope-proportionality`, `project-gotchas`,
  `completeness-check`) collapses into one hook
  `hooks/agent-launch-preamble.sh` emitting ONLY the structural contract
  (`RESULT:`, `TRUST_REPORT:`, `PROGRESS: [step N/M]`) + a single
  bullet-list advisory synthesised from the four sources. Target
  ≤ 400 tokens/launch. Flag: `context.merged_agent_preamble`.

### D3 — Resource consumption

- **MCP lazy-start.** Do not auto-spawn the 8 long-lived MCP servers on
  every session. Mark `scheduled-tasks`, `ccd_session`, `ccd_directory`,
  `plugin_engram`, `claude_preview`, `mcp-registry`, `claude-in-chrome`
  as lazy; keep `engram` eager for now. Target idle MCP RSS ≤ 150 MiB
  (down from ~630 MiB). Flag: `resources.mcp.lazy_start`.
- **JSONL rotation.** `.cognitive-os/metrics/*.jsonl` rotated by size
  (>1 MiB, per ADR-028 D1.A) or age (>7 days); rotated files archived under
  `.cognitive-os/metrics/archive/` for 30 days. Implemented as a
  `SessionStart` hook `rotate-metrics.sh` (no-op if run within last
  hour). Flag: `metrics.rotation.enabled`.
- **Hook dedupe.** Four per-Agent-launch hooks → one (covered by D2).
  Three post-Agent hooks (`auto-verify`, `dod-gate`, `completion-gate`) →
  one `agent-completion-gate.sh` running the regex suites in a single
  python call (not three shell forks). Flag:
  `hooks.merge_agent_completion`.
- **`lean` as default-capable profile.** Promote `lean` to recommended
  default for trivial/small tasks (per `rules/adaptive-bypass.md`).
  `lib/adaptive_profile_selector.py` reads `project.phase` + task-
  complexity hint and picks profile. Flag:
  `efficiency.adaptive_profile`.

## Feature flags (rollback mechanism)

All flags live in `cognitive-os.yaml`. Each defaults to SAFE (current
behaviour) during Phase 0 rollout and flips to the new value only after
its phase lands.

| Flag | Default (post-rollout) | Flipping to `false` restores |
|---|---|---|
| `testing.global_verify_on_stop` | `true` | No global verify at Stop. |
| `testing.targeted_verify` | `true` | Agents go back to `pytest tests/unit` default. |
| `session.silent_self_install` | `true` | Banner emitted on every session. |
| `context.claude_md_compact` | `true` | Legacy full CLAUDE.md loaded. |
| `context.merged_agent_preamble` | `true` | Four-hook quartet restored. |
| `resources.mcp.lazy_start` | `true` | Eager MCP startup restored. |
| `metrics.rotation.enabled` | `true` | Unbounded JSONL growth restored. |
| `hooks.merge_agent_completion` | `true` | 3-hook PostToolUse chain restored. |
| `efficiency.adaptive_profile` | `true` | Forces `standard` profile (current default). |

Any flag flipped to `false` restores current behaviour within one session
restart.

## Implementation phases

Each phase owns a disjoint surface. The "lock owner" column designates
the single writer for that file. No other phase edits that file without a
handoff PR reviewed by the owner.

### Phase 1 — Tests (D1)

**Goal:** agents verify targeted; orchestrator verifies global once at Stop.

**Surface (lock owner = Phase 1):**

- `rules/agent-quality.md` — add the "targeted vs global" section.
- `.cognitive-os/skills/sdd-apply/SKILL.md` — replace literal
  `pytest tests/unit` with `{targeted_test_command}` placeholder.
- `.cognitive-os/skills/sdd-verify/SKILL.md` — same.
- `.cognitive-os/skills/test-driven-development/SKILL.md` — same.
- `.cognitive-os/skills/verification-before-completion/SKILL.md` — same.
- Up to **3 additional** skill manifests flagged in the grep of Baseline
  §"Full-suite default sites".
- `hooks/global-verify.sh` — NEW (`Stop` matcher).
- `lib/targeted_test_resolver.py` — NEW.
- `tests/unit/test_targeted_test_resolver.py` — NEW.
- `.claude/settings.json` — **coordinated edit** (register Stop hook);
  see Parallelism plan.
- `templates/cognitive-os.yaml` — add `testing.*` keys.

**Tasks:**

1. Implement `lib/targeted_test_resolver.py` with mapping rules
   (`lib/foo.py` → `tests/unit/test_foo.py`; `hooks/x.sh` →
   `tests/unit/test_hook_x.py`; fallback to `tests/unit/` if unknown).
   Unit-test it.
2. Author `hooks/global-verify.sh`: check `git log` since session start
   marker; if commits exist, run `python3 -m pytest tests/unit/ -q`;
   append result to `.cognitive-os/metrics/global-verify.jsonl`.
3. Replace literal `pytest tests/unit` in skill manifests with
   `{targeted_test_command}` + a short SKILL-side note on how to resolve it.
4. Add the new section to `rules/agent-quality.md`.
5. Add `testing.*` flags to template `cognitive-os.yaml`.
6. Coordinate a single PR with Phase 3 to register the Stop hook.

**Prerequisite (per ADR-027a §3):**

- **`ws9-test-errors` must be resolved before Phase 1 executes.** With 292
  `pytest` collection errors outstanding, `targeted_test_resolver.py` almost
  always falls back to the full suite, defeating Phase 1's purpose. Orchestrator
  MUST NOT launch the Phase 1 agent until
  `python3 -m pytest tests/unit/ --collect-only -q 2>&1 | grep "error" | wc -l`
  returns `0`. (Status 2026-04-21: ws9 already resolved per
  `.cognitive-os/work-queue.json:122`.)

**Risks & mitigations:**

- *Resolver maps wrong → agent skips a failing test.* Fallback to the
  full suite when resolver returns < 1 test file; Phase 1 ships an
  assertion that every changed file yields ≥ 1 test.
- *Session crash before Stop → no global verify.* Also run on `PreCompact`.
- *Skill-manifest edits conflict with Phase 2/3.* Phase 1 touches only
  the acceptance-criteria blocks of skill manifests; Phase 2/3 do not
  edit those blocks.

**Goal:** SessionStart silent when unchanged; CLAUDE.md compact; preamble slim.

**Surface (lock owner = Phase 2):**

- `~/.claude/CLAUDE.md` — migrate to compact ref-key index.

- `hooks/agent-launch-preamble.sh` — NEW, merged preamble.
- `hooks/blast-radius-detector.sh`, `hooks/scope-proportionality.sh`,

  DEMOTED to library helpers under `hooks/_lib/advisory/`; they keep

- `lib/ref_key_loader.py` — NEW.
- `tests/unit/test_ref_key_loader.py` — NEW.

  The `EXCLUDED_RULES` mechanism in `hooks/self-install.sh` (commit `1ee19a4`) already

- `templates/cognitive-os.yaml` — add `session.*` and `context.*` keys.

1. Implement `lib/ref_key_loader.py` + tests.

4. Convert the four advisory hooks to dual-mode (standalone / `--one-line`).
5. Register the merged hook in `.claude/settings.json`; demote (not
   delete) the four originals.
6. After in-flight self-install work lands, patch banner guard.

- *Merged preamble drops a critical advisory.* Four source hooks remain
  as library helpers; flipping `context.merged_agent_preamble: false`

## Parallelism plan

### Contended files (evidence-based)

Files that two or more phases want to edit:

| File | Phase 1 | Phase 2 | Phase 3 | Declared owner |
|---|---|---|---|---|
| `.claude/settings.json` | registers `global-verify.sh` (Stop) | swaps 4 Agent entries for 1 | swaps 3 PostToolUse entries for 1 | **Phase 3** (largest change). Phases 1 & 2 file PRs against a Phase-3 branch. |
| `scripts/apply-efficiency-profile.sh` | — | adds merged-preamble toggle to profiles | adds merged-completion toggle + adaptive selector | **Phase 3** |
| `templates/cognitive-os.yaml` | `testing.*` | `session.*`, `context.*` | `resources.*`, `metrics.*`, `hooks.*`, `efficiency.*` | **shared**: each phase appends under a distinct top-level key → no structural conflict. Safe to parallelise with alphabetical-key ordering. |
| `hooks/self-install.sh` | — | 3-line banner guard | — | **Phase 2**, BLOCKED until the in-flight agent commits. |

All other files across the three phases are disjoint.

### Recommended: Path A (parallel-safe with one serialisation point)

1. **Phase 3 starts first** and owns the two riskiest files
   (`.claude/settings.json`, `scripts/apply-efficiency-profile.sh`). It
   merges before Phases 1 and 2 land their settings patches.
2. **Phases 1 and 2 run concurrently** after Phase 3's registration
   infrastructure lands. They submit PRs against the settings file;
   Phase 3's owner reviews and merges.
3. Phase 2 defers `self-install.sh` work until the in-flight symlinks
   agent commits. Everything else in Phase 2 is independent.

### Path B (strictly serial)

Phase 3 → Phase 1 → Phase 2. Estimated wall-clock:
Phase 3 (~4 h) + Phase 1 (~3 h) + Phase 2 (~5 h) = **~12 h**.

Path A estimate: Phase 3 (~4 h) + max(Phase 1, Phase 2) (~5 h) = **~9 h**.

### Recommendation: **Path A**

- The risk reduction from front-loading `.claude/settings.json` under a
  single owner is the single most important safety property.
- Phases 1 and 2 have **zero file overlap** with each other; giving up
  parallelism on top of Path A's safety gain is unnecessary.
- Expected savings ≈ 25–30 % wall-clock vs Path B.

## Rollback plan per phase

### Phase 1

- **Commit to revert:** single squash-merge commit `feat(adr-027/phase-1)`.
- **Flag to flip:** `testing.targeted_verify: false` — restores
  `pytest tests/unit` as the default for all agents instantly.
- **Regression test:** `tests/unit/test_targeted_test_resolver.py` must
  pass; a smoke `sdd-apply` run must show the agent did NOT invoke the
  full suite on its own.

### Phase 2

- **Commit to revert:** `feat(adr-027/phase-2)`.
- **Flags to flip:** three independent flags —
  `session.silent_self_install`, `context.claude_md_compact`,
  `context.merged_agent_preamble` — each reverts one sub-change.
- **Regression test:** `scripts/adr-027-measure/session-tokens.sh`
  asserts SessionStart stdout ≤ 50 tokens and per-Agent-launch ≤ 400
  tokens in a smoke session. Failure = rollback.

### Phase 3

- **Commit to revert:** `feat(adr-027/phase-3)`.
- **Flags to flip:** four independent flags —
  `resources.mcp.lazy_start`, `metrics.rotation.enabled`,
  `hooks.merge_agent_completion`, `efficiency.adaptive_profile`.
- **Regression test:** `tests/unit/test_rotate_metrics.py` +
  `tests/unit/test_agent_completion_gate.py` + smoke `ps`-based check

## Appendix B — Uncertainties (honest list)

1. **MCP RSS numbers are approximate (±10–20 %).** macOS `ps -Axo rss`
   includes copy-on-write shared pages. A precise measurement needs
   `vmmap` or `footprint` per PID, which requires user interaction. The
   630 MiB figure should be treated as "roughly this, definitely > 400 MiB".
2. **Hook runtime p50/p95 come from a non-uniform `hook-health.jsonl` schema.**
   Some rows have `duration_ms`, others `elapsed_ms`, some neither.
   ~40 % of rows were skipped. True percentiles could be ±20–30 %.
3. **Per-Agent-launch token count is bounded, not measured exactly.**
   Sources were grepped for here-docs + echo/printf blocks and bytes
   counted. Actual runtime output depends on branches taken. The
   "525–1,175 tokens" range is a conservative envelope. A precise number
   requires a logging hook that tees every hook's stdout for one session
   — out of scope for planning.
4. **"pytest tests/unit" in skill manifests is a grep heuristic.** Some
   skills may construct the command dynamically from config and not
   contain the literal string. The "≥ 7 skills" count is a lower bound.
5. **`self-install.sh` banner size depends on which branch of its logic
   fires.** "300–450 tokens" is the cache-hit branch; cache-miss can
   emit ≥ 2× more. Phase 2 guards the cache-hit branch only.
6. **JSONL rotation math assumes steady-state ingest.** If metric
   verbosity grows (new hooks, more fields), the 20 MiB target may be
   optimistic. The policy is an upper bound regardless.
7. **Adaptive profile selector heuristics are not yet defined.**
   `adaptive-bypass.md` describes intent; the mapping from `(phase,
   task_complexity)` → profile has never been validated. Phase 3 ships
   conservative defaults and collects telemetry for ~2 weeks before tuning.
8. **Compact `~/.claude/CLAUDE.md` may lose a rule the orchestrator
   silently relies on.** The ref-key loader has a fallback, but if the
   orchestrator parses `CLAUDE.md` directly (bypassing the loader), a
   rule could become invisible. Mitigation: one-week observation window.
9. **I did not measure the token cost of ToolSearch first-use inflation.**
   User categorised as "minor, acceptable"; accepted without independent
   measurement.
10. **Settings.json hook count (27) counts entries, not runtime
    invocations.** Several entries share a matcher, so actual per-event
    dispatches per tool call may be lower than 27/total. Target of
    "≤ 18 entries" still improves load but may not directly translate
    to 33 % fewer invocations per call.
