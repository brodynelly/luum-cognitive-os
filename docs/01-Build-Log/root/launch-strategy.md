# Launch Strategy — Cognitive OS

> From building in the dark to shipping in the light.

## Phase 0: Immediate (Today)

- [ ] Stop adding features
- [ ] Create orphan branch (clean history)
- [ ] Write compelling README (outcomes, not features)
- [ ] Record 2-minute demo video (before/after)
- [ ] Publish to GitHub

## Phase 1: First Users (Week 1)

- [ ] Post on Twitter/X with demo
- [ ] Post on Reddit r/ClaudeAI, r/programming
- [ ] Share in Claude Code Discord
- [ ] Ask 5 developer friends to try it
- [ ] Collect feedback (what confused them, what didn't work)

## Phase 2: Iterate (Weeks 2-4)

- [ ] Fix top 3 user complaints
- [ ] Simplify installation based on where users got stuck
- [ ] Add the features users ASKED for (not what we assumed)
- [ ] Write case studies from first users

## Phase 3: Grow (Month 2+)

- [ ] Blog posts showing before/after with real projects
- [ ] YouTube tutorial (5 minutes, not 45)
- [ ] Hacker News launch
- [ ] Product Hunt launch
- [ ] Community contributions guide

## Success Metrics

| Metric | Week 1 target | Month 1 target |
|---|---|---|
| GitHub stars | 50 | 500 |
| Installs (cos init) | 20 | 200 |
| Daily active users | 5 | 30 |
| Issues/PRs from community | 3 | 20 |
| Retention (still using after 1 week) | 40% | 50% |

## Key Insight

The phased launch plan prioritizes learning over building. Each phase produces feedback that shapes the next phase. No phase lasts longer than necessary -- move forward as soon as the learning is captured.
