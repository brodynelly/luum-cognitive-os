# Framework-Specific Context (Example Template)

This is an EXAMPLE template. Projects should create their own framework context.

For a Go project, you might document:
- Monorepo layout and directory conventions
- HTTP framework patterns (Gin, Chi, Fiber, etc.)
- Use case and controller interfaces
- Cross-service communication patterns
- Testing conventions

For a Node.js project:
- Module structure (NestJS modules, Express routes)
- Dependency injection patterns
- TypeScript conventions
- Testing framework setup

Create your project-specific framework context in `.cognitive-os/templates/` or `{project}/.claude/templates/`.
