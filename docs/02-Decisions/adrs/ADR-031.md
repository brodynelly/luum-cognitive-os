# ADR-031 — Continuous Aspirational/Dormant/Real Audit

**Status**: Accepted  
**Date**: 2026-04-20  
**Related**: ADR-028 (observability), rules/ROADMAP.md (hook gap analysis)

## Problem

Manual forensic audits showed a persistent gap between the agentic primitives we build and the agentic primitives
that produce real behaviour. Without continuous measurement, the ratio is invisible and grows silently.
The recurring question "what actually runs?" has no automated answer.

## Decision

Introduce a weekly automated audit (`scripts/aspirational_audit.py`) triggered from a SessionStart
hook (`hooks/aspirational-audit-weekly.sh`). The audit classifies every hook, lib module, script,
and skill into one of four categories using a 6-priority signal ladder.

## Signal Taxonomy (first-match wins)

| Priority | Signal | Result |
|----------|--------|--------|
| 1 | fire_count_7d > 0 AND registered in settings.json | REAL |
| 2 | registered but no fire count | DORMANT |
| 3 | not registered; in EXCLUDED_HOOKS.txt (LIBRARY/DEPRECATED/ADMIN) | METADATA |
| 3b | not registered; in EXCLUDED_HOOKS.txt (FUTURE/CONDITIONAL) | ASPIRATIONAL |
| 4 | writes to an existing metrics JSONL file | REAL (via output) |
| 5 | lib module with ≥1 non-test caller | REAL (via callers) |
| 6 | skill invoked in last 30d per skill-invocations.jsonl | REAL |
| default | code exists, none of the above | DORMANT |

Deprecation shims (< 30 lines + DEPRECATED marker) → METADATA regardless of registration.

## Threshold Policy

If `(DORMANT + ASPIRATIONAL) / total > 40%`, the SessionStart hook emits a stderr advisory
pointing to the dated report in `docs/06-Daily/reports/`. No blocking — fail-open always.

## Outputs

- `.cognitive-os/metrics/aspirational-audit.jsonl` — one MetricEvent row per agentic primitive
- `docs/06-Daily/reports/aspirational-audit-YYYY-MM-DD.md` — human-readable table + summary
- `.cognitive-os/metrics/.last-aspirational-audit` — throttle timestamp

## Rollback

1. Remove `aspirational-audit-weekly.sh` from SessionStart in `.claude/settings.json`
2. Delete `hooks/aspirational-audit-weekly.sh` and `scripts/aspirational_audit.py`
3. Optionally delete generated reports and JSONL rows

## Residual Risk

**False-REAL**: a hook that fires but produces wrong behaviour appears REAL. The audit measures
presence and frequency, not correctness. Correctness remains the responsibility of `sdd-verify`
and adversarial review.
