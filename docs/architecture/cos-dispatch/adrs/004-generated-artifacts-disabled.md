# ADR-004: Generated Artifacts Start Disabled

## Status

Accepted

## Context

The auto-generator creates validators and transformers from detected patterns. If these are enabled by default, a bad pattern detection could block legitimate tool calls or corrupt tool input/output.

Options:
1. Auto-enable all generated artifacts (maximum automation, maximum risk)
2. Start disabled, require human review (safe by default, slower adoption)
3. Auto-enable above a confidence threshold (compromise)

## Decision

All generated artifacts start with `enabled: false` in their config. A human must review and explicitly enable them via `cos-dispatch review`. The CLI lists pending artifacts with their source pattern, confidence, and generated code for inspection.

## Consequences

- Safe by default: a bad pattern detection cannot break the system
- Human-in-the-loop ensures generated code quality
- Slower adoption curve — auto-improvement requires periodic human review
- Over time, as feedback accumulates and confidence calibrates, we can introduce `auto_enable_above_confidence: 0.95` for high-confidence patterns
- The feedback loop (enable/disable/modify/delete) trains the generator to produce better artifacts
- Generated artifacts live in `generated/` directory with clear `AutoGenerated: true` metadata
