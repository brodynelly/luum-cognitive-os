# ADR-041 — Exercised Coverage Pipeline (MVP)

**Date**: 2026-04-20
**Status**: Accepted
**Deciders**: luum-agent-os team
**Supersedes**: none
**Related**: ADR-028 (SLOs), ADR-031 (aspirational-audit)

---

## Problem Statement

As of 2026-04-20, the aspirational-audit reports:

| Classification | Count |
|----------------|-------|
| REAL           | 183   |
| DORMANT        | 580   |
| ASPIRATIONAL   | 126   |
| METADATA       | 68    |
| **Total**      | **957** |

**Dormant+Aspirational ratio**: 706 / (706 + 183) = **79.4%**

An agentic primitive is DORMANT or ASPIRATIONAL when it has no observed invocations in
`.cognitive-os/metrics/*.jsonl` within the session window. At 79.4% this means
~4 out of every 5 hooks, scripts, and skills are dead weight — they silently fail
or never execute, defeating the purpose of the OS.

The goal of this ADR is to drive that ratio below **40%** through a structured
exercise pipeline: classify agentic primitives by risk tier, generate chaos-test skeletons,
and systematically exercise them starting with safety-critical (Tier A) items.

---

## Tier Definitions

### Tier A — Safety-Critical (~30 agentic primitives)

Agentic primitives whose failure or non-invocation directly threatens data integrity,
security, or system safety. These MUST be exercised first.

**Path heuristics** (matched via regex):
```
/blocker/|credential|secret|killswitch|destructive|disk.?full
content.?policy|rate.?limit|auto.?rollback|auto.?refine|auto.?verify
guardrail|license.?guard|concurrent.?write|release.?guard
global.?verify|error.?pipeline|error.?learning
```

**Consequence of non-exercise**: silent failures in blockers can allow destructive
operations, secret leaks, or undetected test regressions to pass through the system.

### Tier B — Infrastructure (~60 agentic primitives)

Daemons, schedulers, scanners, and long-running monitors. Failure is not
immediately data-destructive but causes operational blindness.

**Path heuristics**:
```
monitor|scheduler|daemon|scanner|heartbeat|reaper|executor
registry|capture|session.?init|session.?end|metrics
```

**Consequence of non-exercise**: missing observability data, stale process
tables, broken session hygiene.

### Tier C — Advisory / Feature-Gated (~100 agentic primitives)

MLflow sync, Valkey wiring, onboarding wizards, and other optional integrations
that are behind feature flags or require external services.

**Path heuristics**:
```
mlflow|valkey|onboarding|advisory|audit.?weekly
```

Or content signal: file contains `if [ -z "$OPT_FEATURE"` or `if \[ "\$FEATURE_` patterns.

**Default tier for unclassified agentic primitives.**

### Tier D — Skills + Rules Metadata (~130 agentic primitives)

`SKILL.md` files, rule markdown files, and template assets that have no
cross-environment invocation telemetry. These are exercised by ensuring the
skill infrastructure reads them correctly, not by direct invocation.

**Path heuristics**:
```
skills/.*/SKILL\.md|rules/.*\.md|templates/.*
```

## Classifier Algorithm

The classifier (`scripts/cos_classify_coverage.py`) uses a waterfall of heuristics
applied in tier-priority order (A → B → D → C as default):

```
for primitive in all_dormant_aspirational:
    if matches(tier_a_regex, primitive.path):
        assign Tier A
    elif matches(tier_b_regex, primitive.path):
        assign Tier B
    elif matches(tier_d_regex, primitive.path):
        assign Tier D
    else:
        assign Tier C (default)
```

The classifier reads `.cognitive-os/metrics/aspirational-audit.jsonl` and outputs
`.cognitive-os/coverage-tiers.json` with `{primitive_path: tier}` mapping.

Override mechanism: an agentic primitive with a `TIER_OVERRIDE: A` comment in its first
20 lines will be forced to that tier regardless of path heuristics.

## Chaos Test Template Generator

`scripts/cos_chaos_template.py` generates `tests/chaos/test_{name}_exercised.py`
skeletons with:

1. **Fixture**: `tmp_path` + minimal `.cognitive-os/` directory structure
2. **Trigger**: subprocess invocation of the agentic primitive with synthetic inputs
3. **Assertions**: exit code check + JSONL row emitted OR state change observable
4. **Placeholders**: `# PLACEHOLDER: describe actual trigger condition` sections

Template pattern for hooks:
- Create minimal JSON payload matching the hook's expected stdin format
- Inject appropriate env vars (agent context, killswitch state)
- Assert exit code behavior (0=pass, 1=advisory, 2=block)
- Assert JSONL row written to appropriate metrics file

Template pattern for scripts:
- Invoke via subprocess with `--summary` or equivalent flags
- Assert non-zero output length or specific output patterns
- Assert output file existence

## Wave Rollout Plan

### Wave A — Tier A Safety-Critical (~2 sessions)

Target: 10+ Tier A agentic primitives with verified chaos tests.
Success: All new tests pass on first run. Re-run aspirational-audit shows
DORMANT→REAL transitions for exercised agentic primitives.

Agentic primitives targeted in MVP:
1. `hooks/secret-detector.sh` — PreToolUse redaction + PostToolUse advisory
2. `hooks/content-policy.sh` — prohibited term scanner
3. `hooks/auto-rollback-trigger.sh` — retry-loop exceeded detector
4. `hooks/auto-verify.sh` — acceptance criteria runner
5. `hooks/auto-refine.sh` — failure marker + retry emitter
6. `hooks/concurrent-write-guard.sh` — flock serialization
7. `hooks/release-guard.sh` — manual release blocker
8. `hooks/error-learning.sh` — failure capture + deduplication
9. `hooks/error-pipeline.sh` — merged error detection pipeline
10. `hooks/guardrails-validator.sh` — PII detection (feature-gated)

### Wave B — Tier B Infrastructure (~3 sessions)

Target: 20+ Tier B agentic primitives.
Focus: session lifecycle hooks (session-init, session-end, state-heartbeat),
monitor hooks (agent-bus-monitor, reaper-heartbeat), and capture hooks
(git-context-capture, conversation-capture).

### Wave C — Tier C Advisory (~1 session)

Target: 15+ Tier C agentic primitives via feature-flag activation in test environment.
Focus: mlflow-sync (mock MLflow server), valkey-ensure (mock Valkey),
aspirational-audit-weekly (time-travel test).

### Wave D — Tier D Skills/Rules (Ongoing)

Target: Ongoing. Add skill invocation tests as part of each new skill shipped.
Focus: Ensure `SKILL.md` files are parseable and skill entry points are callable.

## Verification: Ratio Drop Target

After Wave A (10 Tier A agentic primitives exercised):
- New REAL count: 183 + 10 = 193
- Ratio: (706 - 10) / (706 - 10 + 193) = 696 / 889 = **78.3%** (improvement: ~1.1pp)

After Wave A+B (30 components exercised):
- New REAL count: 183 + 30 = 213
- Ratio: (706 - 30) / (706 - 30 + 213) = 676 / 889 = **76.0%** (improvement: ~3.4pp)

After Wave A+B+C (50 components exercised):
- New REAL count: 183 + 50 = 233
- Ratio: (706 - 50) / (706 - 50 + 233) = 656 / 889 = **73.8%** (improvement: ~5.6pp)

Reaching <40% requires exercising ~250+ currently dormant agentic primitives across
all waves. This is an ongoing initiative, not a single sprint.

## Acceptance Criteria

1. `python3 scripts/cos_classify_coverage.py --summary` prints tier counts
2. `.cognitive-os/coverage-tiers.json` exists with all DORMANT+ASPIRATIONAL agentic primitives mapped
3. `ls tests/chaos/*_exercised.py | wc -l` >= 10 (new tests, not counting pre-existing)
4. `pytest tests/chaos/*_exercised.py -v` → all new tests pass (skipif for env-gated tests is acceptable)
5. `pytest tests/unit/test_coverage_classifier.py tests/unit/test_chaos_template.py -v` → 7/7 pass
6. Ratio drop of 1-2pp observable after re-running aspirational-audit
7. This ADR document exists at `docs/02-Decisions/adrs/ADR-041.md`

## Alternatives Considered

**Manual audit + fix**: Too slow at 706 dormant agentic primitives. Systematic tooling is required.

**Integration testing only**: Integration tests require the full Claude Code
runtime. Chaos tests are reproducible in any CI environment via subprocess.

**Coverage.py instrumentation**: Bash hooks cannot be instrumented by Python
coverage tools. Process-level chaos tests (subprocess invocation) are the
only viable approach for shell agentic primitives.

---

## Consequences

**Positive**:
- Safety-critical hooks are verified to behave correctly before production incidents
- Ratio metric is now trackable and actionable
- New agentic primitives ship with chaos test skeletons (force multiplier)

**Negative**:
- Initial overhead: ~5-10 minutes per Tier A agentic primitive to complete skeleton
- Some Tier A agentic primitives require complex environment mocking (skipif acceptable)
- Classifier heuristics will misclassify edge cases; manual overrides needed
