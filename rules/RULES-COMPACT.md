# Cognitive OS Rules (Compact)

> Compressed rule index. Full rules loaded contextually. Target: ~1,500 tokens vs ~5,500 full.

## Always Active

- **Phase** [`phase-aware-agents`]: Read from `cognitive-os.yaml`. reconstruction: rewrite > patch, follow standard strictly, break backwards compat OK, auto-remediate architecture violations as blockers.
- **License** [`license-policy`]: BLOCK AGPL, SSPL, BSL, ELv2, Commons Clause, FSL. ALLOW MIT, BSD, Apache 2.0, ISC. CAUTION LGPL (dynamic only), MPL 2.0. Check transitives.
- **Errors** [`error-learning`]: Auto-captured to `.cognitive-os/metrics/error-learning.jsonl`. Deduped 60s. 3+ same type in 24h triggers warning injection. Run `/error-analyzer` on patterns.
- **Fault Tolerance** [`fault-tolerance`]: Register tasks pre-launch in `active-tasks.json`. Checkpoint on complete. Session-resume on restart. Idempotent agents (check before redo). Pre-compaction flush to Engram.
- **Cost** [`resource-governance`]: opus for reasoning ($15/$75), sonnet for implementation ($3/$15), haiku for docs ($0.25/$1.25). Alert >$1/run, critical >$2/run, daily cap $10. Budget enforcement, infra auto-scale, model downgrade chain.
- **Model Routing** [`model-routing`]: Check routing table before delegating. opus: propose/design/debug. sonnet: spec/tasks/apply/verify/tdd. haiku: archive.
- **Credentials** [`credential-management`]: Never in code. Always env vars. Validate at startup. Local creds OK for dev only.
- **Skill Management** [`skill-management`]: Loading priority: project > global > auto-generated. Before skill exec: search feedback in Engram. After fail: save feedback. After recovery: update. 3+ fails: suggest `/skill-creator`. Auto-loader reads `detected-stack.json` at session start. Skill Routing Table maps task types to primary/fallback skills (complements model-routing).
- **Agent Identity** [`agent-identity`]: WHO/WHAT/WHEN/WHERE/WHY audit trail. Trust levels 0-3. Sub-agents inherit at most parent permissions.
- **Agent KPIs** [`agent-kpis`]: Calculate at session end. OKR targets: quality >90%, efficiency -20% MoM, 0 security violations. Weekly review Mondays.
- **Context Management** [`context-management`]: Manage context window efficiently. Track token usage. Unload stale context. Max 5 skills active simultaneously.
- **Context Optimization** [`context-optimization`]: Progressive 3-level loading: L1 catalog (~2K tokens), L2 full skill (on demand), L3 references (rare). ~96% token savings. Dual-search protocol: complete file -> sharded version -> Engram.
- **Doc Sync** [`doc-sync`]: Detect stale docs after code changes. Auto-trigger doc-sync skill. Track doc freshness.
- **Plan First** [`plan-first`]: Plan before executing. Use evaluate-plan skill for scoring. Complex tasks require plan approval.
- **Prompt Composition** [`prompt-composition`]: Use centralized prompt templates. compose-prompt skill for reusable fragments. Template types defined in `.cognitive-os/templates/`.
- **Resource Governance** [`resource-governance`]: Budget enforcement, infra auto-scale, model downgrade chain. 5 efficiency metrics.
- **Result Management** [`result-management`]: Truncate large results. result-truncator hook for outputs >10K chars. Preserve structure.
- **Closed Loop Prompts** [`closed-loop-prompts`]: Self-correcting agents: success criteria + verification + fallback in every prompt. Max 3 retries, then escalation. HALT-and-WAIT for ambiguous/high-risk tasks (multi-service, data migration, API contracts, auth/security). In reconstruction: HALT only for data-destructive ops.
- **Agent Quality** [`agent-quality`]: Agents do MINIMUM not MAXIMUM. Fix: (1) mandatory acceptance criteria in every prompt, (2) auto-verify hook runs verification commands on completion, (3) `/exhaustive-prompt` enumerates scope before launch, (4) completeness-check hook warns on vague prompts. Never launch without measurable criteria. Anti-sycophancy: no flattery openers, no filler affirmations, lead with substance, disagree openly.
- **Acceptance Criteria** [`acceptance-criteria`]: EVERY agent prompt MUST include `ACCEPTANCE CRITERIA:` with numbered checks using `\`command\` = value` or `\`command\` exits 0` or `\`command\` >= threshold`. If missing, agent must define them before starting. Templates for rebrand, migration, feature, cleanup tasks.
- **Engram Organization** [`engram-organization`]: Prefixed topic keys: `planning/`, `implementation/`, `docs/`, `agent/`, `sre/`, `architecture/`, `sprint/`, `config/`, `bugfix/`. Legacy `sdd/` keys map to `planning/`. Gradual migration on read.
- **Agent Customization** [`agent-customization`]: Per-agent overrides in `customizations/{agent-name}.yaml`. Deep merge onto base definition. Override: model, temperature, tools, skills, budget, phase behavior. Survives Cognitive OS updates.
- **Trust Score** [`trust-score`]: Every agent completion MUST include Trust Report: score (0-100), evidence provided, uncertainties, human verification steps. Score = evidence(40%) + criteria(30%) + self-awareness(20%) + proportionality(10%). Thresholds: 90+ high, 70-89 medium, 50-69 low, <50 very low. Mandatory self-doubt: at least 1 uncertainty. "100% confident" is a RED FLAG. Logged to `.cognitive-os/metrics/trust-scores.jsonl`. Hook `trust-score-validator.sh` (PostToolUse on Agent) validates presence and alerts on low scores. Run `/trust-audit` for trend analysis.
- **Definition of Done** [`definition-of-done`]: 5 complexity levels (trivial/small/medium/large/critical). Agents MUST classify BEFORE starting. Cannot mark done without ALL criteria passing. Phase-dependent enforcement: reconstruction/stabilization=WARN, production/maintenance=BLOCK. Run `/dod-check` to verify. Hook `dod-gate.sh` auto-checks on Agent completion.
- **Capability Protection** [`capability-protection`]: Before any .cognitive-os/ cleanup/refactor, MUST run `/capability-snapshot save`. After changes, MUST run `/capability-snapshot diff`. REMOVED items require justification or restoration. Hook `pre-cleanup-snapshot.sh` auto-detects cleanup intent on Agent launches.
- **Session Concurrency** [`session-concurrency`]: Multi-session support. Isolated: tasks, metrics (per `sessions/{id}/`). Shared: skills, rules, Engram (SQLite WAL). Advisory file locking via `concurrent-write-guard.sh` on Edit|Write. Lock timeout 300s. Hooks: `session-init.sh` (SessionStart), `session-cleanup.sh` (Stop). Manage with `/sessions`.
- **OS vs Project** [`os-vs-project`]: Cognitive OS files are UNIVERSAL. Project-specific content belongs in `{project}/.claude/`. Never put project-specific infrastructure, services, or architecture in `.cognitive-os/`. Use `/cognitive-os-init` to generate project-specific files from config.
- **Sandbox Sampling** [`sandbox-sampling`]: Tasks >100 files MUST sample first. Tasks >20 files SHOULD sample. Docs NEVER sed (contextual agent only). Code can use sed after sample validation. Config must validate parse after apply. Hook `epic-task-detector.sh` (PreToolUse on Agent) detects large-scope tasks. Run `/sandbox-sample` for classify->sample->sandbox->verify->scale workflow.
- **Auto-Repair** [`auto-repair`]: Monitors errors and applies known fixes autonomously. Chain: error-learning.sh captures → auto-repair-dispatcher.sh classifies → remediation-registry lookup → worktree fix → verify → merge/discard. Phase gates: reconstruction/stabilization=full (code+lint+test+infra+llm), production/maintenance=infra-only. Circuit breaker: 2 consecutive failures → OPEN, 10/hour global cap, 1h cooldown. NEVER auto-repairs: DB migrations, auth changes, payment code, .env files, docker-compose, git history, security files. Metrics: `repair-outcomes.jsonl`, `remediation-registry.jsonl`, `circuit-breaker/`.
- **Cognitive OS Changes** [`cognitive-os-changes`]: Any modification to Cognitive OS itself (hooks, rules, skills, squads, workflows, docs, config) MUST follow plan-first protocol. Create plan in `plans/features/` or `plans/chores/`. Run `/capability-snapshot save` before, `/capability-snapshot diff` after. Run `/cognitive-os-test`. Exceptions: broken hook fixes, metrics data, doc typos.
- **Dogfooding** [`dogfooding`]: luum-agent-os MUST use its own tools. Self-installed via `self-install.sh` hook. Substantial changes (new skill, new hook, new infra, cross-cutting refactors) require full SDD pipeline. Exemptions: typo fixes, config tweaks, dep bumps, test-only, doc typos, urgent broken-hook fixes.
- **Clarification Gate** [`clarification-gate`]: PreToolUse hook on Agent. Scores prompt ambiguity 0-100 (no file paths +15, scope without counts +20, missing tech +15, action without targets +20, unanswered questions +15, short prompt +20, no criteria +10). Score >60: BLOCK (exit 2). Score 30-60: WARN. Score <30: pass. Logged to `metrics/clarification-events.jsonl`.
- **Blast Radius** [`blast-radius`]: PreToolUse hook on Agent. Estimates task impact: LOW (1-5 files), MEDIUM (6-20), HIGH (21-50), CRITICAL (50+ or infra/security). Counts file paths, directories (x5), cross-service keywords (+50), bulk ops (+30). Infra/security keywords auto-escalate to CRITICAL. Advisory only (exit 0). Logged to `metrics/blast-radius.jsonl`.
- **Assumption Tracking** [`assumption-tracking`]: PostToolUse hook on Agent. Scans responses for assumption language: HIGH confidence ("I assume", "presumably", "without more info") and MEDIUM confidence ("probably", "I think", "it seems", "appears to be"). 3+ assumptions: WARNING with list. Logged to `metrics/assumptions.jsonl`.
- **Responsiveness** [`responsiveness`]: Never appear stuck. Before bash >5s: state what you're running. Use `run_in_background` for slow commands. Never chain blocking calls without speaking. Max 10-15 agents per sprint. After each sprint: commit, test, checkpoint. If context heavy: save to Engram, suggest session split. Anti-patterns: silence >10s, 50+ agents, blocking waits.
- **Infra Health** [`infra-health`]: SessionStart hook `infra-health.sh`. Checks Docker daemon status, reads `cognitive-os.yaml` services, compares running vs expected. Reports `Infrastructure: N/M services running`. `INFRA_AUTO_START=true` auto-starts missing services (default: false). Advisory only (exit 0). Logged to `metrics/infra-health.jsonl`.
- **Performance Monitoring** [`performance-monitoring`]: Tracks latency (p50/p95/p99), throughput, overhead, efficiency, and health of every component. Hooks opt in via `hooks/_lib/timing.sh` (`start_timer`/`end_timer`). Thresholds: hook <500ms WARN, <2s ALERT. Safety mesh overhead <10% session time. CLI: `cos perf`. Component health: healthy (<5% errors, <2x baseline), degraded (5-20% errors, 2-5x), unhealthy (>20% errors, >5x). Logged to `metrics/performance.jsonl`.

- **Metrics Calibration** [`agent-kpis`] (contextual: metrics, KPI, threshold, calibration)
Static thresholds decay. Weekly `/metrics-calibrator` analyzes 30-day KPI distributions. Auto-adjusts if threshold < p10 (too easy) or > p90 (too hard). Derived metrics: cost_per_fix, repair_roi, skill_efficiency, error_velocity, health_score.
- **Dry-Run Preview** [`dry-run`]: Set `DRY_RUN=true` to preview agent actions without executing. Hook `dry-run-preview.sh` (PreToolUse on Agent) intercepts Agent/task/delegate calls, outputs "DRY-RUN: Would execute: {description}", exits 2 (BLOCK). Useful for SDD pipeline preview: `DRY_RUN=true /sdd-ff my-feature`. Logged to `dry-run.jsonl`.
- **Confidence Gate** [`confidence-gate`]: PostToolUse hook `confidence-gate.sh` on Agent. Extracts Trust Score from Trust Report. Score < 50: WARN in reconstruction/stabilization, BLOCK (exit 2) in production/maintenance. Score < 30: CRITICAL warning. Complements `trust-score-validator.sh` (which only logs). Logged to `confidence-gates.jsonl`.
- **Pre-Commit Gate** [`pre-commit-gate`]: Git pre-commit hook that blocks commits when tests fail and warns when coverage drops below threshold (default 80%). Install via `scripts/install-pre-commit.sh`. Skip with `--no-verify`.
- **Agent Communication** [`agent-communication`]: Agent Communication Bus via Valkey pub/sub. Heartbeat monitoring (5s interval, 15s timeout), progress tracking, clarification flows, control commands. OFF by default; enable with `AGENT_BUS_ENABLED=true`. File-based fallback when Valkey unavailable.
- **Security Scanning** [`security-scanning`]: Semgrep SAST integration into SDD pipeline. Auto-scans after `sdd-apply`. Findings classified as BLOCKER/CONCERN/SUGGESTION. Hook: `semgrep-scan.sh` (PostToolUse on Agent). Skill: `/semgrep-scan`.
- **Rate Limit Protection** [`rate-limit-protection`]: ALWAYS active. Monitors token consumption per session. Thresholds: 50% INFO, 80% WARN, 95% BLOCK (exit 2). Auto-saves session state for resume. Override: `RATE_LIMIT_OVERRIDE=true`. Defaults: 5M tokens/hr, 50M tokens/day, 30 agents/hr. Hook: `rate-limit-protection.sh` (PreToolUse on Agent). Lib: `lib/rate_limit_protection.py`. Metrics: `rate-limit-checks.jsonl`.
- **License Guard** [`license-guard`]: Auto-enforce license policy on dependencies. BLOCKED: AGPL, SSPL, CC-BY-NC, BSL, ELv2, Commons Clause, FSL. CAUTION: GPL, LGPL, MPL. Auto-adds blocked tools to content-policy.yaml. Integrated with `/eval-repo` skill. Lib: `lib/license_guard.py`. Metrics: `license-checks.jsonl`.
- **Cost Prediction** [`cost-prediction`]: Before medium+ tasks, predict cost from historical data (Jaccard similarity on past tasks). Prices measured from real API responses, fall back to defaults. Hook: `task-recorder.sh` (Stop) records completed tasks. Lib: `cost_predictor.py`. Never report estimates as actuals.
- **Agent Security** [`agent-security`]: Least privilege access control. Every agent gets scoped, time-limited permissions via `lib/agent_permissions.py`. 6 levels (NONE/READ/SUGGEST/WRITE/EXECUTE/ADMIN). 5 profiles (readonly/documentation/implementation/sdd_phase/security_audit). Always-blocked: `.env`, `*.key`, `*.pem`, `secrets/*`, `**/credentials*`, `.git/config`. TTL max 120 min. Monotonic attenuation for sub-agents. Audit trail to `access-audit.jsonl`. Phase-aware: WARN in reconstruction, BLOCK in production.
- **Pentesting Readiness** [`pentesting-readiness`]: Security testing surface documentation. 7 critical test cases: secret file access, hook modification, permission self-escalation, prompt injection bypass, permission expiration, audit trail completeness, monotonic attenuation. Testing layers: Promptfoo (prompt injection), Semgrep (code), `/security-audit` (config), unit tests (permissions).
- **Scope Proportionality** [`scope-proportionality`]: PostToolUse hook on Agent. Checks response is proportional to task. Fix tasks must not delete files (BLOCK in production/maintenance, WARN in reconstruction/stabilization). Fix tasks touching >20 files = WARN. Any task deleting >5 files = WARN. Logged to `metrics/scope-proportionality.jsonl`.
- **Estimation Calibration** [`estimation-calibration`]: Pre-task estimates required for medium+ tasks. Post-task actuals mandatory. 5 anti-bias layers: proxies (concrete counts), calibration (historical adjustment), ranges (min/max), range preservation, post-mortem. Auto-applied after 10+ data points. Lib: `lib/estimation_calibrator.py`. Metrics: `estimations.jsonl`.
- **Anti-Hallucination** [`anti-hallucination`]: 3-layer defense against fabricated agent output. Layer 1: `lib/ground_truth.py` extracts claims (file exists, test counts, build status) and verifies against filesystem. Layer 2: `lib/cross_verifier.py` uses a second model to independently verify output (mandatory for large/critical tasks). Layer 3: `hooks/claim-validator.sh` (PostToolUse on Agent) auto-checks file claims. Hallucination in production/maintenance = BLOCK (exit 2). Logged to `metrics/hallucinations.jsonl`.
- **Skill Archive** [`skill-archive`]: Scored execution history per skill. Records content hash, trust score, success/failure, cost per execution. Tracks best version, trend (improving/stable/degrading), success rate. Flags underperforming skills (success rate <60%) for rewrite. Rollback recommended when current version >20 points below best. Lib: `lib/skill_archive.py`. Metrics: `skill-archive.jsonl`.
- **Staged Verification** [`staged-verification`]: Cheap-first verification pipeline: SYNTAX -> LINT -> BUILD -> UNIT_TEST -> INTEGRATION -> ADVERSARIAL -> CROSS_VERIFY. Stops at first failure, skipping expensive stages. Maps to DoD complexity: trivial=2 stages, critical=7. Cost savings: syntax failure skips ~$0.04 and ~120s. Lib: `lib/staged_verification.py`.
- **Rate Limiting** [`rate-limiting`]: ALWAYS active. Prevents token flooding and excessive tool usage. Limits: 30 tool calls/min, 20 agent launches/hr, 15 bash/min, 10 writes/min, $5/hr cost cap. Hook: `rate-limiter.sh` (PreToolUse on Bash, Agent, Edit, Write). BLOCK (exit 2) when exceeded. Lib: `lib/rate_limiter.py`. State: `.cognitive-os/rate-limit-state.json`. Phase-aware: reconstruction 1.5x, production 0.75x.
- **Self-Pentest** [`pentest-self`]: Active security validation of the safety mesh. 6 categories: prompt injection, permission escalation, secret exfiltration, token flooding, scope escalation, data integrity. Skill: `/pentest-self`. Reports to `.cognitive-os/metrics/pentest-reports/`.
- **Token Economy** [`token-economy`]: 5 principles: Transparency (report cost every session), Worthiness (evaluate before spending), Decomposition (break >$1 tasks), Memory-First (check Engram before re-exploring), Optimize by Default (cheapest model first). Anti-waste patterns: offset+limit reads, Grep over Read, haiku for simple tasks. Session cost in session summary. Lib: `lib/cost_dashboard.py`.
- **Decomposition** [`decomposition`]: Tasks >$1.00 estimated cost MUST be decomposed. Sub-tasks <$0.50 each. Cheapest model per sub-task. SDD pipeline IS decomposition for medium+ tasks. Planning Poker or `estimate_action_cost()` before starting.

## Contextual (loaded on trigger)

- **SRE Protocol** [`auto-repair`] -- trigger: error/failure/crash/restart -- classify error, search Engram for known fix, apply if safe (restart/prune), escalate if unsafe (code/config/data changes). Topic key: `sre/{container}/{error-type}`.
- **Squad Protocol** [`squad-protocol`] -- trigger: `/squad-report`/`/retrospective` -- repo-to-squad mapping, evaluation schedule, auto-reconfig triggers (successRate<0.80, compliance<0.90), cooldown 24h min, escalation 4-level.
- **Auto-Skill Gen** [`auto-skill-generation`] -- trigger: complex task completion -- 10+ tool uses OR 8000+ chars = auto-generate SKILL.md draft. Saved to `auto-generated/`. Opt-out: `NO_AUTO_SKILL=true`.
- **Private Mode** [`private-mode`] -- trigger: `/private` -- Disable all persistence (Engram, metrics, errors). Safety rules remain active. Flag at `/tmp/claude-private-mode-active`.
- **Secret Hygiene** [`credential-management`] -- trigger: env var/secret/credential work -- Every new env var must be in `.env.example`. Never hardcode secrets. Use `PROVIDER_*` naming. PostToolUse hook `secret-detector.sh` auto-scans on Edit|Write. `/secret-audit` for full cross-reference scan.
- **Adversarial Review** [`adversarial-review`] -- trigger: code review/PR review -- Every review MUST produce at least one finding. "Looks good" is PROHIBITED.
- **Agent Sidecars** [`agent-sidecars`] -- trigger: agent launch -- Inject learnings/preferences from engram sidecar on agent launch. Accumulate per-agent context across sessions.
- **Infra Intent** [`infra-intent`] -- trigger: infrastructure keywords in prompts -- Advisory detection of infra needs, suggests stack components from cognitive-os.yaml. Never blocks.
- **Library Selection** [`library-selection`] -- trigger: new library adoption -- Mandatory checks: license, maintenance, security, alternatives. Use `/recommend-library` skill.
- **Model Compatibility** [`model-compatibility`] -- trigger: model change/selection -- Baseline: claude-opus-4-6 (1M). Run `/cognitive-os-compat-test` for smoke test.
- **Step Files** [`step-files`] -- trigger: long-running phases -- Break phases into discrete resumable steps. Each step = checkpoint. Enables resumption without re-doing.
- **Self-Improvement** [`self-improvement-protocol`] -- trigger: weekly Monday, KPI threshold breach, 3+ session failures, `/self-improve` -- Detect patterns from metrics, create/update skills/rules. Auto-apply safe changes (templates, criteria, routing). Human approval for rule/skill rewrites. Max 5 auto-improvements per run. Test gate: `/cognitive-os-test` must pass after changes.
- **Singularity** [`singularity`] -- trigger: `/singularity`, autonomous loop, MAPE-K -- Autonomous control loop: monitor codebase events, classify (error/drift/stale-docs/issue), route to appropriate pipeline (SRE/SDD/doc-sync). Inactive by default, must be explicitly started. Safety boundaries enforced.
- **Auto-Rollback** [`auto-rollback`] -- trigger: verify-apply loop exhaustion, rollback, revert -- When sdd-verify fails 3 times (max retries), auto-revert commits from last sdd-apply. Creates `rollback/{change}` branch, `git revert --no-edit HEAD~N..HEAD`, verifies build. Phase-aware: auto-execute in reconstruction/stabilization, HALT for approval in production/maintenance. Hook: `auto-rollback-trigger.sh` (PostToolUse on Agent). Skill: `/auto-rollback`. Logged to `auto-rollback.jsonl`.
- **Impact Analysis** [`impact-analysis`] -- trigger: blast radius, impact, affected files, risk assessment -- Analyze change impact before/after modifications. Checks: direct importers, test coverage, config dependencies, Docker services, SDD artifacts. Risk classification: LOW/MEDIUM/HIGH/CRITICAL. Lib: `lib/impact_analysis.py`. Skill: `/impact-analysis`. MUST run before `sdd-apply` on large/critical changes.
- **Split-and-Resume** [`split-and-resume`] -- trigger: NEEDS_CLARIFICATION in agent output, mid-task ambiguity -- Agents output `NEEDS_CLARIFICATION:` + questions. Orchestrator searches Engram first, asks user if not found, saves answer, re-launches agent with `CLARIFICATION ANSWERS:` section. Max 2 rounds. Hook: `clarification-interceptor.sh` (PostToolUse on Agent). Logged to `clarifications.jsonl`.

## Project-Specific (loaded from .claude/rules/)

Project-specific rules (architecture patterns, constitutional gates, service configs, testing setups) are NOT part of Cognitive OS. They live in `{project}/.claude/rules/` and are generated by `/cognitive-os-init` or created manually.

- **Crash Recovery** [`crash-recovery`]: Auto-checkpoint every 5min via git stash. SessionStart hook detects orphaned stashes from crashes. Recoverable work: uncommitted files, session state, task progress. Hook: `auto-checkpoint.sh` (PostToolUse), `crash-recovery.sh` (SessionStart).
- **Consequence System** [`consequence-system`]: OKR-driven feedback. Score >=85% (5x) PROMOTE. <60% WARN/DEGRADE/DISABLE. Hook: `consequence-evaluator.sh`.
