## Root Cause: Tracker Blind Spots

The tracker has 4 structural blind spots that explain all 14 lib false positives and the 35 rule false positives:

### Lib tracker blind spots

1. **`from lib import X` pattern**: Tracker only matches `from lib.X import` / `import lib.X`. Misses: `from lib import file_lock_registry`, `from lib import orchestrator_mode`.
2. **sys.path imports in tests**: Many package libs are tested via `sys.path.insert(0, lib_dir)` then `from batch_runner import`. Tracker pattern requires `lib.` prefix.
3. **`lib/` cross-imports**: Tracker searches hooks/skills/tests/templates but NOT lib/ itself. Misses: `lib/model_router.py` importing `lib/litellm_client`.
4. **`packages/*/hooks/` and `packages/*/lib/`**: Tracker doesn't search these dirs. Misses: packages/verification-audit/hooks/result-truncator.sh importing smart_truncator.

### Rule tracker blind spots

1. **EXCLUDED_RULES not scanned**: The tracker checks RULES-COMPACT backtick refs and hooks that reference `rules/X.md` files. It does NOT scan `hooks/self-install.sh:EXCLUDED_RULES` which explicitly accounts for all 35 unreferenced rules.
2. **Section C (contextual rules)**: Rules intentionally omitted from RULES-COMPACT to keep context lean — they load on demand. The tracker treats absence from RULES-COMPACT as "unreferenced".
